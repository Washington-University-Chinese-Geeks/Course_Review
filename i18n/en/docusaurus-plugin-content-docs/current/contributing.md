# Contributing

Thanks for helping WUCG Course Review grow — your review will help every student who comes after you.

## How contributions work

All content changes go through **GitHub Pull Requests**. A maintainer reviews and merges; GitHub Actions then deploys the updated site.

1. Fork [Washington-University-Chinese-Geeks/Course_Review](https://github.com/Washington-University-Chinese-Geeks/Course_Review).
2. Create a branch: `git checkout -b review/cse-2407-your-name`.
3. Edit or create the course `.mdx` file (see below).
4. Open a PR describing what you changed.
5. Sourcery will automatically generate an English translation of any new Chinese content in the PR.

> Chinese (`docs/`) is the **source of truth**. Only edit Chinese files for content. English translations in `i18n/en/` are machine-generated by Sourcery on each PR — you can edit them directly if a translation is off.

## Adding a review to an existing course

Each reviewed course lives at `docs/<dept>/reviews/<code>.mdx`. Append a new `<CourseReview>` block:

```mdx
<CourseReview
  reviewer="Anonymous / Your name"
  professor="Professor name"
  semester="Fall 2025"
  content={`Objective facts: what the class covers, assignment style, exam format...`}
  evaluation={`Subjective take: teaching quality, workload, grading curve...`}
  tips={`Practical tips: pick this prof's section, buy a used kit, etc.`}
/>
```

**Style notes**:

- Put objective facts ("the course teaches Java") in `content`.
- Put opinions ("the professor is a great lecturer") in `evaluation`.
- Use `tips` for logistical advice and common pitfalls.

## Creating a page for an un-reviewed course

Many Math and Econ courses don't have reviews yet. To add the first one, create `docs/<dept>/reviews/<code>.mdx`:

```mdx
---
title: CSE 4XXX — Course Title
sidebar_label: CSE 4XXX
---

# CSE 4XXX: Course Title

<CourseInfo
  code="CSE 4XXX"
  title="Course Title"
  oldCodes={["CSE 3XX"]}
  lastOffered="Spring 2026"
  link="https://bulletin.wustl.edu/..."
  description="Official course description."
/>

<CourseReview
  reviewer="..."
  professor="..."
  content={`...`}
  evaluation={`...`}
  tips={`...`}
/>
```

Then update the corresponding department `index.mdx` so the `<CourseCard>` links to your new page and flags `hasReview`.

## Local development

```bash
npm install
npm start         # Chinese site (default locale)
npm run start:en  # English preview
npm run build     # Production build
```

## Code of conduct

- Keep reviews constructive. Criticism of a professor should focus on teaching, not on the person.
- Do not upload exam questions, answers, or anything that violates WashU Academic Integrity.
- Multiple reviews per course are welcome — different semesters and different professors add value.
